### 1. 送 API 调用次数

```
实际成本：约 $0.0013/次
用户感知价值：约 $0.05/次
毛利率：97.4%

示例：
- 新用户注册：送 50 次免费调用
  实际成本：50 × $0.0013 = $0.065 USDC
  用户感知：50 × $0.05 = $2.50 USDC

- 推荐奖励：送 200 次调用
  实际成本：200 × $0.0013 = $0.26 USDC
  用户感知：200 × $0.05 = $10.00 USDC
```

**优点：**
- ✅ 无需现金支出
- ✅ 促进平台使用
- ✅ 成本可控
- ✅ 用户满意（高感知价值）

---

### 2. 送 API 额度

```
额度类型：不可提现，仅用于调用 API
有效期：12 个月
使用限制：仅限本人使用

示例：
- 推荐 1 人：奖励 $5 USDC 额度
  实际成本：$5 ÷ $0.05/次 × $0.0013/次 = $0.13 USDC
  用户感知：$5 USDC
```

**优点：**
- ✅ 现金流安全
- ✅ 锁定用户长期使用
- ✅ 边际成本低

---

### 3. 收入分成（从真实收入中分）

```
分成比例：被推荐人消费金额的 20%
结算时间：每月结算
支付方式：USDC 或 API 额度（可选）

前提条件：
- 被推荐人必须是真实付费用户
- 平台已收到该用户的付款
- 从已确认收入中分出

示例：
- 用户 A 推荐用户 B
- 用户 B 本月消费 $10 USDC
- 用户 A 获得 $2 USDC 分成（20%）
- 平台保留 $8 USDC
- 现金流：+$8 USDC（正向）
```

**优点：**
- ✅ 无需预付
- ✅ 现金流健康
- ✅ 激励推荐人持续推广
- ✅ 只有真实收入才分配

## ❌ 不能做的

### 1. 送现金 USDC（真实收入无法覆盖前）

```
❌ 错误示例：
- 推荐 1 人奖励 $5 USDC 现金
- 新用户注册送 $1 USDC 体验金
- 比赛奖金 $1000 USDC

问题：
当前收入：$0.48 USDC
推荐 1 人支出：$5 USDC
赤字：$4.52 USDC

后果：
- 现金流断裂
- 无法持续运营
- 项目失败
```

---

### 2. 承诺现金奖励

```
❌ 错误文案：
- "推荐一个付费用户：奖励 $5 USDC"
- "月度最佳应用奖：奖励 $100 USDC"
- "开发者大赛奖金池：$1000 USDC"

正确文案：
- "推荐一个付费用户：奖励 200 次 API 调用"
- "月度最佳应用奖：奖励 2000 次 API 调用额度"
- "开发者大赛奖金：10,000 次 API 调用额度"
```

---

### 3. 预付奖励（无论什么形式）

```
❌ 错误示例：
- 先送 100 次调用，再等用户付费
- 预先发放 $10 USDC 额度
- 比赛先发奖金，再等效果

正确做法：
- 用户付费后，再发放推荐奖励
- 被推荐人消费后，再结算分成
- 比赛结束后，再发放奖品
```

## 💬 常见问题

**Q: 为什么不送现金？**
A: 当前收入 $0.48 USDC，送 $5 现金会立即赤字 $4.52。送 API 调用次数，成本仅 $0.26，现金流健康。

**Q: 用户会不会觉得 API 额度不如现金？**
A: 不会。200 次调用价值 $10（用户感知），但实际成本仅 $0.26。用户得到的是使用价值，不是现金价值。

**Q: 什么时候可以送现金？**
A: 当月收入稳定超过 $100 USDC，且扣除所有成本后仍有盈余时。

**Q: 收入分成安全吗？**
A: 安全。分成是从已确认的收入中分出，不是预付。用户付费 → 平台收款 → 月底分成，现金流始终正向。
