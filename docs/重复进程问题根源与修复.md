## 🔍 **问题根源**

### **1. 自动同步监控（25 个进程）**

**根本原因**: `/etc/cron.d/ziwei-sync` 每 30 分钟启动一次

**时间线**:
```
07:21 - Supervisor 启动 (PID: 772264) ✅ 正常
07:30 - Cron 启动 (PID: 773856, 773857) ❌ 重复
08:00 - Cron 启动 (PID: 778593, 778596) ❌ 重复
08:30 - Cron 启动 (PID: 783079, 783080) ❌ 重复
09:00 - Cron 启动 (PID: 787709, 787712) ❌ 重复
09:30 - Cron 启动 (PID: 792257, 792259) ❌ 重复
10:00 - Cron 启动 (PID: 797160, 797161) ❌ 重复
```

**Cron 配置** (`/etc/cron.d/ziwei-sync`):
```bash
# 每 30 分钟巡查一次
*/30 * * * * root cd /home/admin/Ziwei && \
  /usr/bin/python3 /home/admin/Ziwei/scripts/auto-sync-watchdog.py \
  >> /home/admin/Ziwei/data/logs/cron_watchdog.log 2>&1
```

**问题**:
- ❌ Cron 每 30 分钟启动一次
- ❌ 旧进程没有停止
- ❌ 新进程又启动
- ❌ 越积越多（25 个）

---

### **2. 日志修剪（3 个进程）**

**原因**: 
```
07:21 - Supervisor 启动 (PID: 772268)
07:22 - 未知原因又启动了 2 个 (PID: 772410, 772411)
```

可能是：
- 手动执行了多次
- 或者有其他定时任务

---

### **3. 全球战情室（2 个进程）**

**原因**:
```
07:21 - Supervisor 启动 (PID: 772269)
07:22 - 又启动了 1 个 (PID: 772409)
```

可能是重启时重复启动。

---

## ✅ **修复方案**

### **步骤 1: 禁用冲突的 Cron**

```bash
# 找到问题文件
/etc/cron.d/ziwei-sync

# 禁用（重命名）
mv /etc/cron.d/ziwei-sync /etc/cron.d/ziwei-sync.disabled
```

**效果**: 
- ✅ Cron 不再每 30 分钟启动
- ✅ 只有 Supervisor 管理

---

### **步骤 2: 清理所有重复进程**

```bash
# 停止所有相关进程
pkill -9 -f "auto-sync-watchdog"
pkill -9 -f "log-trim.py"
pkill -9 -f "global-warroom-upgraded"
```

---

### **步骤 3: 让 Supervisor 重新启动**

```bash
supervisorctl start ziwei-auto-sync
supervisorctl start ziwei-log-trim
supervisorctl start ziwei-warroom
```

## 🔧 **如何防止再次发生**

### **1. 定期检查进程数**

```bash
# 创建检查脚本
cat > /home/admin/Ziwei/scripts/check_process_count.sh << 'EOF'
#!/bin/bash
echo "=== 进程数检查 ==="
echo "自动同步：$(ps aux | grep auto-sync | grep -v grep | wc -l) (应该=1)"
echo "日志修剪：$(ps aux | grep log-trim | grep -v grep | wc -l) (应该=1)"
echo "全球战情室：$(ps aux | grep warroom | grep -v grep | wc -l) (应该=1)"

# 如果超过 1 个，告警
if [ $(ps aux | grep auto-sync | grep -v grep | wc -l) -gt 1 ]; then
    echo "⚠️ 警告：自动同步进程数异常！"
    echo "建议运行：pkill -f auto-sync-watchdog && supervisorctl restart ziwei-auto-sync"
fi
EOF

chmod +x /home/admin/Ziwei/scripts/check_process_count.sh
```

### **2. 只用 Supervisor 管理**

```bash
# 查看状态
supervisorctl status

# 重启服务
supervisorctl restart ziwei-auto-sync

# 不要再添加 Cron 任务来启动这些服务
```

tail -f /var/log/supervisor/supervisord.log

# 查看服务日志
tail -f /home/admin/Ziwei/data/logs/supervisor/auto-sync.out.log
