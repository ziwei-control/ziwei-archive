# 服务修复记录 - 2026-03-04

## 问题描述

1. **Dashboard UI** ❌ FATAL - Exited too quickly
2. **自动同步监控** ⚠️ 显示已停止（实际运行正常）
3. **全球战情室** 重复进程 (2 个)
4. **日志修剪** 重复进程 (3 个)

## 根本原因

### Dashboard 问题
- 端口 8081 被手动启动的进程占用
- Supervisor 无法绑定端口，启动失败
- 循环重启导致 FATAL 状态

### 重复进程问题（核心问题）
- **systemd 服务** 和 **Supervisor** 同时管理相同服务
- `/etc/systemd/system/` 下有 ziwei-warroom.service 等
- `/etc/supervisor/conf.d/ziwei.conf` 也有相同配置
- 导致每个服务启动 2 个进程

## 修复步骤

### 1. 修复 Dashboard
```bash
# 停止手动进程
pkill -9 -f dashboard_v3_framer.py

# 重启 Supervisor 服务
supervisorctl restart ziwei-dashboard
```

### 2. 解决 systemd 与 Supervisor 冲突
```bash
# 停止并禁用 systemd 服务
systemctl stop ziwei-warroom ziwei-log-trim log-trim
systemctl disable ziwei-warroom ziwei-log-trim log-trim

# 清理重复进程
pkill -9 -f global-warroom-upgraded
pkill -9 -f "log-trim.py watch"

# 重启 Supervisor 服务
supervisorctl restart ziwei-warroom ziwei-log-trim
```

### 3. 验证服务状态
```bash
supervisorctl status
# 所有服务应为 RUNNING 状态
```

## Dashboard CLI 修复详情

### 问题
OpenClaw CLI 交互终端无法发送消息

### 根本原因
1. JavaScript 模板字符串 `${command}` 与 Python `str.format()` 冲突
2. `do_POST` 方法中 `json.dumps({{` 语法错误（双括号应为单括号）

### 修复方案
```python
# 1. JavaScript 模板字符串改为字符串拼接
${command} -> ' + command + '
${data.error} -> ' + data.error + '
${error.message} -> ' + error.message + '

# 2. Python 代码修复
json.dumps({{ -> json.dumps({
}}).encode -> }).encode

# 3. CSS 部分转义（第 24-620 行）
{{ -> {{{{
```
